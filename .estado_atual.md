# Estado atual — 2026-07-15

## Tarefa ativa (Gerente)

**Bug de produto:** tours de onboarding/tutorial
1. Disparam **sem organizacao ativa** (usuario sem `ngp:active-org` / membership).
2. **Reaparecem** ao reentrar na pagina mesmo apos o usuario ja ter visto/fechado o tour.

**Impacto:** friccao no 1o uso; overlay bloqueia UX em shell vazio; sensacao de "nao lembra" que o usuario ja leu.

## Diagnostico (codigo vs spec)

| Spec (`docs/50-components/onboarding-tour.md`) | Codigo hoje | Gap |
|-----------------------------------------------|-------------|-----|
| Auto global na 1a visita autenticada | `notifyShowWorkload` → `startGlobalTour` sem checar org | Sem gate de org ativa |
| Auto page apos global, se incompleto | `tryAutoStartPageTour` + `localStorage` | Sem gate de org; persistencia fragil se tour quebra (alvos `data-tour` ausentes → Driver pode nao concluir → `mark*Completed` nao grava) |
| Persistencia browser: `ngp:onboarding-tour-completed` / `ngp:page-tour-completed:{id}` | `onboarding-tour-storage.ts` / `page-tour-storage.ts` | Marca so em `onDestroyed`; se tour falha/remonta sem destroy limpo, chave nunca vira `"1"` → loop |
| Shell tem `orgId` no loader | `ShellCacheData` **nao** expoe `activeOrgId` ao client | Provider nao consegue gatear |

**Causa raiz provavel (negocio → tecnico):**
- Sem org, paginas/sidebar ficam sem alvos → tour inicia mesmo assim → experiencia quebrada e/ou nao persiste → reabre sempre.
- Reabertura = `is*TourCompleted() === false` (storage nao gravado), nao "reset intencional".

## Decisao de negocio (travar)

1. **Auto-start** (global + por pagina) **SO com organizacao ativa** (`activeOrgId` nao-nulo).
2. Sem org ativa: **nao** abrir tour automatico; botao manual em `/help` / "Ver tour desta pagina" pode permanecer (opcional: desabilitar se sem org — preferencia: desabilitar auto apenas).
3. Ao **fechar (X), Esc, Concluir ou destruir** o tour: **sempre** marcar completed (nao so "terminou todos os passos").
4. Se alvos do tour nao estiverem no DOM apos timeout curto: **nao** iniciar (ou marcar skipped/completed para nao loopar).
5. Persistencia continua **localStorage por browser** no MVP (spec ja diz; sem Supabase).

## Escopo MVP deste bugfix
