## Decisões TRAVADAS

| # | Decisão |
|---|----------|
| 1 | Modelo Permissions → Presets → Users (nunca permission solta no usuário). |
| 2 | Checks runtime por **permission code**; papéis/presets só empacotam codes. |
| 3 | União: **maior poder vence** (membership direto + bypass org + dept). |
| 4 | Teto monday-style: custom preset ⊆ ceiling do papel-base (Administrador do projeto). |
| 5 | Sem papel **comment-only** até existir capability real. |
| 6 | Enums DB **não** renomear (`admin`/`manager`/`viewer`/`owner`). |
| 7 | Owner/Admin org **cria/edita** presets; Admin projeto (`members.*`) **atribui**. |
| 8 | Grupos = atalho opcional (depois de MVP + presets). |
| 9 | **Catálogo fino** `board.cards.*` / `columns.*` / `members.*` etc.; aliases `edit_content` / `manage_members`. |
| 10 | UI: checklist unico (settings + invite); Selecionar tudo/Limpar **no fim** de cada grupo. |
| 11 | RLS: `can_write_board` / `can_manage_board_members` = OR via alias; UI/actions = codes finos. |
| 12 | Fora: grupos de usuarios, F.2 field-level, comment-only. |
| 13 | **Members UI preset-first**: lista mostra `presetName`; select = catálogo; assign por `preset_id`. |

## Fase MVP (papéis fixos)

### Organização

| DB | Label UI | Poder |
|----|----------|-------|
| `owner` | Proprietário | Identidade + membros/convites + write/ACL global |
| `admin` | Administrador | Membros/convites + write/ACL global |
| `manager` | Gerente | Operação; write via ACL/dept (sem convites org) |
| `viewer` | Visualizador | Leitura |

- `app.can_manage_org_members` = **owner \| admin** (não manager).

### Projeto (board)

| DB | Label UI | Poder |
|----|----------|-------|
| `manager` | Administrador | Write + ACL + settings |
| `admin` | Editor | Write **sem** ACL |
| `viewer` | Visualizador | Read |

### Departamento

| DB | Label UI | Poder |
|----|----------|-------|
| `manager` | Gerente | Membros + write dept |
| `admin` | Administrador | Write dept |
| `viewer` | Visualizador | Leitura |
